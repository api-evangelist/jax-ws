# JAX-WS (jax-ws)

JAX-WS (Java API for XML Web Services) is a Java standard for building and consuming SOAP-based XML web services. Originally specified as JSR 224 under the Java Community Process, JAX-WS is now part of Jakarta EE as Jakarta XML Web Services. It defines annotations and runtime APIs that allow developers to expose Java methods as SOAP operations and to generate Java client stubs from WSDL documents.

## Standards & Implementations

### Jakarta XML Web Services (JAX-WS)

The Jakarta EE specification for XML web services, formerly JSR 224. Defines annotations such as `@WebService`, `@WebMethod`, and `@WebParam`, plus runtime APIs for SOAP service providers and clients.

- [Specification](https://jakarta.ee/specifications/xml-web-services/4.0/)
- [Documentation](https://jakarta.ee/specifications/xml-web-services/)
- [GitHub](https://github.com/jakartaee/xml-web-services)

### Eclipse Metro

Reference implementation of Jakarta XML Web Services (JAX-WS), providing a high-performance, extensible SOAP stack.

- [Documentation](https://eclipse-ee4j.github.io/metro-jax-ws/)
- [GitHub](https://github.com/eclipse-ee4j/metro-jax-ws)

### Apache CXF

Open source services framework supporting JAX-WS and JAX-RS for SOAP and REST web services.

- [Documentation](https://cxf.apache.org/docs/index.html)
- [GitHub](https://github.com/apache/cxf)
